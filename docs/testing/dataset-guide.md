# 测试数据集使用说明

## 推荐数据集

- `AliMeeting`

适合你们这个项目，因为它本身就是中文会议场景语音数据。

## 使用建议

不要一开始下载完整数据集做训练或全量测试。

当前阶段只建议做两件事：

1. 先抽取少量音频做功能测试
2. 后续再挑几段多人会议音频做演示

## 推荐取样方式

### 方案一：只取少量样本

目标：

- 3 到 5 个短音频
- 1 到 2 个多人会议音频
- 1 个带明显关键词的音频

下载后放到：

- `sample-data/audio/`
- `sample-data/expected/`

## 推荐测试文件组织

```text
sample-data/
├─ audio/
│  ├─ short-clear.wav
│  ├─ meeting-multi-speaker.wav
│  └─ keyword-demo.wav
└─ expected/
   ├─ short-clear.txt
   ├─ meeting-multi-speaker.txt
   └─ keyword-demo.txt
```

## 你现在最需要做的事

### 1. 先下载数据集

可以优先从公开镜像查看：

- OpenSLR: `https://www.openslr.org/119/`

### 2. 只选一小部分音频

优先选：

- 单段较短的会议音频
- 带对应文本标注的音频
- 多人说话但不要太长的音频

### 3. 手工整理预期文本

即使原数据集有标注，你也最好再人工整理一版简化文本，方便后续做：

- 转写结果对比
- 摘要效果检查
- 关键词命中检查

## 当前阶段不要做的事

- 不要全量下载后直接塞进仓库
- 不要把大音频文件直接提交到 GitHub
- 不要急着做训练

## Git 规范

大体积原始数据不要提交到仓库。

仓库里建议只保留：

- 数据说明文档
- 小样本文件
- 人工整理的预期文本

如果单个音频较大，就只在本地保存，并在文档里记录文件名和来源。
